Qwen Character Translator

GitHub Pages теперь публикует только безопасную информационную страницу. Рабочий интерфейс переводчика отдается локальным Gateway через Cloudflare Quick Tunnel и открывается по same-origin адресу `/app/`.

GitHub README Security Deploy

Как запускать сейчас

Новый поток запуска

  1. Запустите setup.bat один раз, чтобы подготовить зависимости и локальный translator.config.json.
  2. При необходимости выполните authorize_qwen.bat и завершите вход в Qwen.
  3. Запустите start_translator.bat.
  4. Локальное окно покажет QR и короткоживущую ссылку вида https://<quick-tunnel>/connect#code=....
  5. Эту ссылку можно открыть на нескольких устройствах в пределах короткого TTL и лимита claims, например на ПК и телефоне.
  6. После успешной привязки браузер откроет same-origin приложение по адресу /app/.

Что изменилось

  • GitHub Pages больше не хранит и не исполняет клиент для Gateway API.
  • В браузер больше не попадает bearer token, который можно читать из localStorage.
  • Сессия создается одноразовым pairing code и живет в HttpOnly cookie.
  • Запросы к API требуют валидную сессию, CSRF header и допустимый origin.

Локальная конфигурация

Что остается только на вашей машине

translator.config.json теперь считается локальным файлом, создается из translator.config.example.json и не коммитится. Runtime URL туннеля, сессии Qwen, cookies и переводимые тексты не публикуются через GitHub Pages.
